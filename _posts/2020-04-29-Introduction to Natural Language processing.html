Backus-Naur Form

As part of learning a programming language, we learn it's grammar. For example, consider the following line of code in python:

print("Hello World")

It has grammar/ rules. The rules are that strings should be within quotes and if we want to print a statement, we should enclose them in parenthesis (Python3).

Backus-Naur invented the form of writing the language rules precisely, and takes the following form:

<non-terminal> -> replacement

<non-terminal> represents something that's unfinished, which can be replaced by a set of non-terminals or terminals. Using the rule, we keep replacing the non-terminals with replacements until we are only left with only terminals.

The languages we use have similar grammatical rules. While humans can understand each other even if the language strays from the correct grammatical rules, computers cannot. How can this be circumvent?

  • process words and phrases to identify keywords, Parts of Speech, named entities and quantities, among others
  • extract relevant parts based on the above process
  • Analyze overall document
